# 星露谷 NPC 闲置微动作设计

## 背景

当前 Stardew NPC 主链路已经把“主模型决策”和“本地执行器”分开：主模型负责人格、任务和目标，本地小模型只适合承担低风险、可限制、可回退的执行职责。跨地图导航证明了一点：让本地小模型推理地图、坐标或日程是不合适的，但这不代表它没有价值。

NPC 在原版日程中经常长时间站立，例如在家、喷泉、商店、广场等地点。这个空档适合让本地小模型做极轻量的“闲置微动作”，增强 NPC 看起来仍在观察和反应的感觉，同时不改变原版日程、不接管主 agent 任务。

## 目标

第一版只做“看起来活着”的低风险表现，但不要停留在单个表情或单次转头。参考 mod 证明 Stardew 里已经有更鲜活的 idle animation / pose / fidget 做法，所以第一版应包含“短动画层”。

- 头顶表情。
- 左右/上下转头，以及短的 `look_around` 转头序列。
- 原地小动作，例如轻微跳一下、轻微抖动、短暂停顿。
- 允许列表里的短 idle animation / idle pose，一次播放后回到标准 idle frame。

第一版明确不做碎碎念、不生成短文字、不发私聊、不触发手机消息。

## 非目标

- 不干涉原版日程。
- 不移动 NPC 到新 tile。
- 不做跨地图导航。
- 不发 `stardew_speak`。
- 不进入私聊或手机线程。
- 不生成文字气泡或长句。
- 不改变友情、礼物、交易、任务状态。
- 不让本地小模型自由调用游戏 API。

## 架构原则

第一版采用：

`主模型决定可尝试 idle_micro_action -> 本地小模型选择枚举动作 -> 宿主/bridge 安全门 -> SMAPI 呈现`

本地小模型只允许从固定枚举里选，不允许自由生成动作、坐标、文本或 API 调用。

建议第一版动作枚举：

- `emote_happy`
- `emote_question`
- `emote_sleepy`
- `emote_music`
- `look_left`
- `look_right`
- `look_up`
- `look_down`
- `look_around`
- `tiny_hop`
- `tiny_shake`
- `idle_pose`
- `idle_animation_once`

`idle_animation_once` 只能选择宿主配置里的允许列表别名，例如 `idle_tinker`、`idle_clean`、`idle_stretch`，不能让本地小模型直接吐 frame index 或任意 `Data/animationDescriptions` key。动画结束必须按朝向回到标准 idle frame，避免 NPC 卡在怪姿势。

如果某些原版 emote id、动作 API 或动画描述在 Stardew 1.6 下不可用，bridge 必须结构化返回 `unsupported_action`，不能静默成功。

## 工具契约草案

父层 intent：

```json
{
  "action": "idle_micro_action",
  "reason": "idle ambience",
  "idleMicroAction": {
    "kind": "emote_happy|emote_question|emote_sleepy|emote_music|look_left|look_right|look_up|look_down|look_around|tiny_hop|tiny_shake|idle_pose|idle_animation_once",
    "animationAlias": "optional allowlisted alias",
    "intensity": "subtle",
    "ttlSeconds": 2
  },
  "speech": {
    "shouldSpeak": false
  }
}
```

executor-only tool：

```json
{
  "npcId": "Haley",
  "kind": "idle_animation_once",
  "animationAlias": "idle_stretch",
  "reason": "idle ambience",
  "ttlSeconds": 2
}
```

禁止字段：

- `text`
- `message`
- `dialogue`
- `destination`
- `locationName`
- `x`
- `y`
- `frame`
- `frameIndex`
- `rawAnimationId`

## 动作分层

第一版动作分成三层，让效果比“表情 + 单次转头”更鲜活，但仍保持安全：

1. `bubble`：只做 icon emote，不做文字。对应 `emote_happy/question/sleepy/music`。
2. `body_fidget`：不改 tile 的身体小动作。对应 `look_around`、`tiny_hop`、`tiny_shake`。
3. `idle_animation`：允许列表里的短动画/姿态。对应 `idle_pose`、`idle_animation_once`。

暂不纳入第一版：

- `wander`：参考里成熟，但会改变 tile，和“不干涉日程、不移动 NPC”冲突。
- `sit/swim`：`The Stardew Squad` 有参考，但依赖位置、家具、水域状态，第一版先不做。
- `talk/think bubble`：参考很多，但用户明确不要碎碎念和文字。

## 安全门与冷却

默认冷却：

- 每个 NPC 默认 10 秒左右。
- 第一版建议配置范围：8-15 秒，围绕 10 秒微调即可。
- 第一版默认 `10s`，不要拉成长冷却；后续如需更慢节奏再单独评估。

执行前必须满足：

- 游戏世界 ready。
- NPC 可解析，且在有效 location。
- NPC 没有 active move command。
- NPC 没有正在进行的 speak/private chat/open menu 交互。
- 不在 event / cutscene / festival / dialogue menu 中。
- NPC 当前不是 sleeping / invisible / being warped / removed。
- 玩家在同地图且 NPC 在当前 viewport 附近，第一版不对不可见 NPC 做表现。
- `idle_animation_once` 必须命中 allowlist，且总时长不超过 2-3 秒。
- 动画结束、被打断、玩家离远、NPC 开始移动时，必须 `Halt()` 并回到当前朝向标准 idle frame。
- 禁止 looping idle animation 作为第一版默认行为；如配置中存在 loop，也必须有硬超时。

如果不满足，返回结构化 skipped：

- `cooldown`
- `npc_busy`
- `world_not_ready`
- `not_visible`
- `menu_blocked`
- `event_blocked`
- `unsupported_action`
- `animation_not_allowed`
- `animation_too_long`

## 第一版验收标准

- 本地小模型能在 idle tick 中选择一个枚举微动作。
- 同地图可见 NPC 空闲时，能看到表情、转头序列、原地小动作或短 idle animation。
- 默认冷却约 10 秒，不会每 tick 刷屏。
- 私聊、移动、原版日程、跨地图导航不受影响。
- 不出现碎碎念、文字气泡或手机消息。
- 动画播放完能回到标准 idle frame，不残留奇怪姿势。
- `stardew_idle_micro_action` 不进入 parent autonomy 默认工具面，只进入 local executor tool surface。
- 日志能解释每次为什么展示或跳过。

## 推荐实现顺序

1. 先做 `emote_*`、`look_*`、`look_around`、`tiny_hop`、`tiny_shake`，全部不改 tile。
2. 再做 `idle_animation_once` 的 allowlist 和动画结束回 idle frame，参考 `The Stardew Squad`。
3. 如果体验稳定，再考虑 NPC 个性化权重，例如 Haley 更常 `emote_happy/look_around/idle_pose`，Penny 更常 `emote_question/idle_animation_once`。
4. 第二版再评估 `wander`、`sit`、`swim` 这类更强表现；它们有参考，但会碰位置/场景条件。
5. 后续再讨论是否允许短文字碎碎念；第一版不做。
