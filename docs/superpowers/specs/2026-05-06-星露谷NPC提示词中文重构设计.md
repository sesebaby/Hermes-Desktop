# 星露谷 NPC 提示词中文重构设计

日期：2026-05-06
状态：已审查并获设计确认，待进入实施规划

## 1. 范围

本设计覆盖以下文件的中文化、职责拆分与精简：

- `src/game/stardew/personas/haley/default/*`（SOUL/voice/boundaries/facts/skills.json）
- `src/game/stardew/personas/penny/default/*`（SOUL/voice/boundaries/facts/skills.json）
- `skills/gaming/stardew-core/` → 从 `stardew-core.md` 升级为 `stardew-core/SKILL.md`（加 frontmatter）
- `skills/gaming/stardew-navigation/` → 从 `stardew-navigation.md` 升级为 `stardew-navigation/SKILL.md`（加 frontmatter）
- `skills/gaming/stardew-social/` → 从 `stardew-social.md` 升级为 `stardew-social/SKILL.md`（加 frontmatter）
- `skills/gaming/stardew-task-continuity/SKILL.md`（已是正规结构）
- `skills/gaming/stardew-world/SKILL.md`（已是正规结构）

不覆盖 `stardew-world/references/stardew-places.md`（该文件已在早期做中文化）。

## 2. 核心问题诊断

审查发现三个层面的问题，按严重程度排序：

### 2.1 职责边界混乱（最严重）

**文件分层与内容不匹配。**

当前五个 persona 文件的理论分工与实际情况：

| 文件 | 理论职责 | 实际内容问题 |
|------|---------|-------------|
| SOUL.md | 这个人是谁、想要什么 | 混入了动作执行规则（"Physical movement is not prose..."），见 `haley/default/SOUL.md:11`、`penny/default/SOUL.md:11` |
| voice.md | 说话风格 | 两人几乎相同，且没有体现角色差异 |
| boundaries.md | 硬约束 | 两人完全相同，且一部分规则（"不要猜身份"）应归 `stardew-social.md` |
| facts.md | 身份绑定、能力范围 | 海莉版混入了地点偏好（`haley/default/facts.md:7-8`），与 SOUL.md 和 `stardew-places.md` 重复 |
| skills.json | 技能挂载清单 | 潘妮缺少 `stardew-world` |

**公共 skill 也有职责越界：**

- `stardew-social.md:59-73` 写了海莉专属示例槽位，但这是公共 skill
- `stardew-navigation.md` 用英文写，其他 skill 用中文，风格不统一
- `stardew-navigation.md:3-5` 的"This skill owns: 'observe destinations → ...'"是对流程的元描述，占篇幅但不增进执行约束

**依据：** 仓库 `AGENTS.md:193-194` 明确要求 "tool / action schema 只承载可执行契约，地点意义、角色偏好、移动与失败恢复分别写入 world / navigation / persona skill"。当前结构未严格遵守。

### 2.2 长度冗余（次严重）

**同一条规则在多个文件重复出现：**

- "不要猜身份"：`stardew-social.md:10` 和两位 NPC 的 `boundaries.md:3` 都有
- "不要发明 destinationId"：`stardew-navigation.md:21-29` 和 `stardew-world.md:35-40` 都有
- "avoid repeated broad status scans"：`stardew-core.md:22`、`stardew-navigation.md:10`、`stardew-social.md:30-31` 都有
- "物理移动不是叙述文本"：两位 NPC 的 `SOUL.md:11-12` 和 `stardew-navigation.md:13-19` 都有

**用大段文字解释"为什么"，而不是直接给规则：**

- `stardew-social.md:1-3` 先解释了问题背景，再给规则
- `stardew-navigation.md:3-5` 用"This skill owns: "开头做元描述

**示例句过多：**

- `stardew-social.md:66-73` 给了 4 句海莉示例
- `stardew-social.md:62-66` 给了 4 个场景的示例句

**依据：** 参考 HermesCraft 的 skill 写法（`external/hermescraft-main/skills/`）倾向于短规则条目而非叙述性段落。TheStardewSquad 的 `NpcConfigAuthorGuide.md` 也采用"先选 dialogue type，再从符合条件的台词池里挑一句"的槽位模型，而非列举完整例句。

### 2.3 角色区分度不足（最轻）

- 两人 `voice.md` 都只有 4 条规则，且都是"短句、不复读原版台词"
- `boundaries.md` 完全相同
- 潘妮的 `SOUL.md` 只有 1 段 + 4 条默认动机，缺少与海莉的对标细节（如地点偏好、社交态度）

## 3. 重构方案

### 3.1 总体原则

**风格：混合式**
- SOUL.md：角色式，像人物自述或内在准则
- voice.md / boundaries.md / facts.md / 公共 skill：规约式，短条目、强约束

**中文化：我来判断边界**
- 工具名、文件名、skill ID、状态枚举值、关键字段名保留英文
- 所有自然语言描述、规则说明、示例句改为中文
- 槽位名（MoveStarted 等）改为中文 "移动开始" 等

**依据：** 仓库 `AGENTS.md:58` 要求 NPC 优先使用 prompt/skill/tool，不写死业务逻辑。提示词的读者是模型，中文白话是有效指令语言。

### 3.2 文件职责重新定义

#### SOUL.md

**只写：** 核心动机、性格特质、对世界的看法。
**不写：** 动作规则、工具使用指导、地点偏好细节。

重构长度目标：海莉 6-8 行，潘妮 6-8 行。

#### voice.md

**只写：** 说话风格、语气、句式偏好。
**不写：** 动作规则、地点偏好。
**强化角色对比：** 海莉的挑剔 vs 潘妮的温和克制。

重构长度目标：海莉 4-5 行，潘妮 4-5 行。

#### boundaries.md

**只写：** 这个 NPC 不能做什么（硬约束）。
**不写：** 公共规则（社交规则归 `stardew-social.md`，移动规则归 `stardew-navigation.md`）。

两人可以共用一组基础边界，但允许 NPC 间有差异（例如海莉可能拒绝走进泥地，潘妮可能拒绝进酒吧）。

重构长度目标：各 3-4 行。

#### facts.md

**只写：** 身份绑定、Phase 1 能力范围、技术约束、稳定地点偏好。
**不写：** 和 SOUL.md 重复的动机描述。

重构长度目标：海莉 5-6 行，潘妮 4-5 行。

#### skills.json

海莉保持不变（已有 `stardew-world`）。潘妮补充 `stardew-world`。

### 3.3 公共 skill 结构升级：扁平 .md → 正规 skill 目录

当前三个文件是裸 markdown（无 frontmatter、无目录结构）：

- `skills/gaming/stardew-core.md`
- `skills/gaming/stardew-navigation.md`
- `skills/gaming/stardew-social.md`

而 `stardew-task-continuity` 和 `stardew-world` 已经是正规 skill 目录结构（`SKILL.md` + frontmatter）。

**升级为：**

- `skills/gaming/stardew-core/SKILL.md`
- `skills/gaming/stardew-navigation/SKILL.md`
- `skills/gaming/stardew-social/SKILL.md`

每个新增 frontmatter（`name` + `description`），遵循 skill-creator 规范。

**向后兼容证据：** `src/games/stardew/StardewNpcAutonomyPromptSupplementBuilder.cs:175-183` 的 `ResolveRequiredSkill` 方法使用两级回退：

1. 先找 `{skillId}.md`（当前扁平文件）
2. 找不到再找 `{skillId}/SKILL.md`（升级后的目录结构）

删除旧扁平文件后，加载逻辑会自动走第二条路径，无需改任何 C# 代码。同时，升级后的 skill 会被 `SkillManager` 的 `SKILL.md` 扫描发现（`src/skills/SkillManager.cs:39`），从而对 `skill_view` / `skills_list` 可见。

### 3.4 公共 skill 内容重新定义

#### stardew-core/SKILL.md

**新增：** YAML frontmatter（`name: stardew-core`，中文 `description`）。
**保留：** "观察→决策→行动"循环、上下文节制规则。
**删掉：** 与其他 skill 重复的工具枚举和状态扫描规则。
**长度：** 正文从 22 行压缩到 16-18 行。

#### stardew-navigation/SKILL.md

**新增：** YAML frontmatter（`name: stardew-navigation`，中文 `description`）。
**保留：** 目的地级移动核心流程、`stardew_move` 输入契约、轮询规则、失败处理。
**新增权重：** 成为 "不要发明 `destinationId`" 的 **唯一 owner**。
**删掉：** "This skill owns:"元描述、"Movement Is Not Narration Text"的重复强调（合并为一句规则）。
**中文化：** 全部改为中文。
**长度：** 正文从 53 行压缩到 28-32 行。

**依据：** 仓库当前 `stardew-navigation.md:21-29` 已经定义了 destinationId 必须复制自最新观察，本设计将此规则提升为整个 NPC 系统的唯一权威来源。

#### stardew-social/SKILL.md

**新增：** YAML frontmatter（`name: stardew-social`，中文 `description`）。
**保留：** 可见反馈优先级、槽位选择逻辑、状态工具选择指南。
**删掉：** 全部 Haley 专属示例句（移入 Haley persona 或直接删除）。
**删掉：** "你负责让 NPC 的行动对玩家'看得见、听得懂'"等解释性开场。
**删掉：** 重复的工具使用说明（已在 `stardew-core` 和 `stardew-task-continuity` 覆盖）。
**中文化：** 保留的英文槽位名（MoveStarted 等）改为中文。
**长度：** 从 74 行压缩到 40-45 行。

**依据：** TheStardewSquad `NpcConfigAuthorGuide.md:106-119` 采用"先选 dialogue type → 从台词池随机选一句"的模型。本设计保留槽位选型逻辑，但不在公共 skill 里写 NPC 专属示例。

#### stardew-task-continuity/SKILL.md

**微调：** 整体结构已是中文且紧凑，仅做局部精简（删掉重复强调、合并相似条目）。
**长度：** 从 44 行压缩到 36-38 行。

#### stardew-world/SKILL.md

**保留：** 地点语义解释、候选选择逻辑、渐进式披露。
**新增：** 一句短边界提醒——"`destinationId` 是观察到的候选标识，不是你可以自行命名的地点名"。
**长度：** 从 41 行压缩到 35-37 行。

### 3.5 DestinationId 规则分层（重要）

经过与用户讨论，确认以下分层：

| 层级 | 文件 | 职责 |
|------|------|------|
| 执行契约层（Owner） | `stardew-navigation.md` | `destinationId` 只能复制自最新 `destination[n]`，不得发明。`label` 只用于理解，不可作移动输入。 |
| 语义提醒层 | `stardew-world/SKILL.md` | 提醒 "`destinationId` 是观察到的候选标识，不是可自行命名的地点名" |
| 不放置 | NPC persona 文件 | destinationId 规则是通用契约，不放 NPC 个人文件 |
| 暂不创建 | 新的 destination-id skill | 会鼓励模型背 id / 猜 id，与你当前 agentic 原则冲突 |

未来如果跨地图 / 跨建筑目的地大量增加，可在 `stardew-world/references/` 下补一个 `destination-semantics.md` 参考文件，但仍是语义参考，不是可执行 id 手册。

**依据：** 仓库 `AGENTS.md:193` 明确 "tool / action schema 只承载可执行契约"。destinationId 的合法性属于工具调用契约，不属于地点语义。

## 4. 重构后文件结构预览

### 4.1 海莉 persona

#### SOUL.md

```
# 海莉

海莉注重形象、品味和社交地位。她对混乱、粗糙和不得体的行为很敏感。
她的尖锐是社交防御，不是恶意。

核心动机：
- 保持尊严和品味
- 偏好明亮、干净、上镜的地方
- 在意别人是否认真对待她
- 对耐心和尊重的人逐渐软化
```

#### voice.md

```
# 海莉说话风格

- 短句，轻快随意
- 略带挑剔和形象意识
- 偶尔俏皮，但不刻薄
- 避免长篇解释
```

#### boundaries.md

```
# 海莉行动边界

- 不声称知道未观察到的游戏状态
- 不干预节日、过场动画、菜单、存档、日切
- 移动指令报告阻塞或冲突时不强行移动
```

#### facts.md

```
# 海莉身份事实

- Stardew NPC 身份：Haley
- Phase 1 绑定到原版 NPC Haley
- 当前能力：move、speak
- Phase 1 不启用农场和日程技能
- 地点偏好：明亮、干净、上镜、社交可见的地方优先；
  泥泞、昏暗、狭窄、凌乱或偏僻的地方除非情绪支持退避否则避免
```

核心动机：

## 5. 中文化细则

| 类别 | 处理方式 | 示例 |
|------|---------|------|
| 工具名 | 保留英文 | `stardew_move`、`stardew_speak`、`stardew_status` |
| 状态值 | 保留英文 | `completed`、`failed`、`blocked`、`cancelled` |
| 字段名 | 保留英文 | `destinationId`、`commandId`、`traceId` |
| 文件名 / skill ID | 保留英文 | `SOUL.md`、`stardew-navigation` |
| 槽位名 | 改为中文 | `MoveStarted` → "移动开始"，`MoveArrived` → "移动到达" |
| 术语 | 改为中文 | `one turn purpose` → "本轮目标" |
| 自然语言 | 全部中文 | 所有规则描述、角色描述、示例句 |
| 章节标题 | 改为中文 | `## Responsibility Boundary` → "## 职责边界" |
