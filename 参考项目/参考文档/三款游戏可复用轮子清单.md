# 三款游戏可复用轮子清单

更新时间：2026-03-26

这份清单只看 `M1` 基础版，目标是少造轮子，先把三款游戏都做成能跑通的首发链路。

## 一眼结论

- `星露谷物语`：现成轮子最成熟，邮件、对话、条件触发都不用从零造
- `环世界`：原生 API 和前置框架最强，重点是自己把 `气泡 + 掉落 + 实例态` 拼起来
- `鬼谷八荒`：最重要的是官方模组编辑器、官方示例和官方文档包，不要先假设有大把开源底层框架

- 官方示例 `〖官方示例〗八九玄功-残卷`

直接复用建议：

- 对话：优先复用原版对话框 / 多人头像式对话框
- 物品发放：优先走奇遇 / 事件 `产出物品`
- 玩家入包：优先走“直接给玩家”，不要先做 NPC 背包中转
- 名称 / 描述：当前最稳是先在对话层和奖励层展示；背包 tooltip 实例级覆写暂不当 `M1` 硬门槛

## 当前最短主链路

- `星露谷物语`：NPC 气泡 / 对话 -> 条件命中 -> 邮件发礼物 -> `modData` + tooltip patch
- `环世界`：NPC 气泡 -> 条件命中 -> `ThingMaker` 创建礼物 -> `GenPlace` 掉地 -> `ThingComp` 挂实例态
- `鬼谷八荒`：NPC 对话提示 -> 奇遇 / 事件 `产出物品` -> 玩家直接获得并入包

## 当前不要做的事

- 不要在 `M1` 给三款游戏统一造一套平台级物品系统
- 不要在 `M1` 追求三款游戏都做到一模一样深的 UI
- 不要在证据不足时承诺“所有游戏都能做背包内实例级名称 / 描述永久覆写”
