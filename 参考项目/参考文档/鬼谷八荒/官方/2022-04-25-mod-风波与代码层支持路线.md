# 2022-04-25 MOD 风波与代码层支持路线

## 来源

- TapTap 转述制作人致歉信：
  https://www.taptap.cn/moment/267685134713488056

## 关键内容摘录

- 官方承认当时的 MOD 路线引发了玩家不满。
- 制作人明确表示：
  - 会关闭 fanbook 跳转并继续优化
  - `官方编辑器将尽快更新，MOD 作者可以修改代码层面的技术支持`
  - 这样做是为了避免非官方编辑器制作的 MOD 失效，并提升未来 MOD 创作扩展性

## 对本项目的价值

- 这是最关键的“官方承认代码层扩展支持”证据之一。
- 对我们当前能力判断：
  - `对话`：间接支撑
  - `记忆`：中等支撑。说明可以自己做状态逻辑，但不是现成记忆 API 文档
  - `创造物品`：中等支撑。说明后续若编辑器原生命令不够，可以考虑代码层补强

## 设计启示

- 不要把鬼谷八荒只看成“改表工具”。
- 与其先假设必须做复杂外挂，不如优先调查：
  - 编辑器剧情命令
  - 产出物品命令
  - 可写代码的补充扩展点
