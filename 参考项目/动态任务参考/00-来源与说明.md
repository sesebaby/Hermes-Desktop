# 动态任务参考来源与说明

整理时间: 2026-03-25

本目录收集的是“动态任务 / 多步骤任务 / 数据驱动任务”方向的开源参考实现。

## 已下载源码仓库

1. `quest-system`
   - 来源: `https://github.com/shomykohai/quest-system`
   - 方向: Godot 4 资源化任务系统

2. `advanced-quest-system-example`
   - 来源: `https://github.com/shomykohai/advanced-quest-system-example`
   - 方向: Godot 4 多步骤任务示例，组合式 step

3. `godot-questify`
   - 来源: `https://github.com/TheWalruzz/godot-questify`
   - 方向: Godot 4 图结构任务编辑器与运行时

4. `scripted-quests`
   - 来源: `https://github.com/TeamMonumenta/scripted-quests`
   - 方向: JSON 驱动的组件式任务系统

5. `QuestFramework`
   - 来源: `https://github.com/purrplingcat/QuestFramework`
   - 方向: Stardew Valley MOD 任务框架

6. `CustomNPCs_1.12.2-Unofficial`
   - 来源: `https://github.com/BetaZavr/CustomNPCs_1.12.2-Unofficial`
   - 方向: Minecraft NPC / 任务 / 多目标任务扩展

7. `Cataclysm-DDA`
   - 来源: `https://github.com/CleverRaven/Cataclysm-DDA`
   - 方向: 开放世界动态任务、任务目标生成、任务链

8. `StardewMods-QuestEssentials`
   - 来源: `https://github.com/purrplingcat/StardewMods`
   - 说明: `QuestEssentials` 源码位于仓库内的 `QuestEssentials/` 子目录
   - 方向: Quest Framework 扩展，多阶段 / 多目标任务

## Git 元数据处理

已移除每个仓库中的 `.git` 目录，当前目录保留的是纯源码快照。

说明:
- 我只移除了 `.git` 仓库元数据。
- `.gitignore`、`.gitattributes`、`.github` 这类项目文件没有删，因为它们有时对理解工程结构和构建方式仍有价值。

## 额外说明

1. `Quest Essentials`
   - 公开页面: `https://www.nexusmods.com/stardewvalley/mods/8827`
   - 源码位置: 已纳入 `StardewMods-QuestEssentials/QuestEssentials`
   - 备注: 它不是独立仓库，而是 `StardewMods` 单仓库中的一个项目。

## 建议阅读顺序

1. 先看 `advanced-quest-system-example`
   - 最贴近 Godot 多步骤任务最小可用实现。

2. 再看 `godot-questify`
   - 适合理解分支、并行目标、任务图。

3. 再看 `quest-system`
   - 看底层资源化任务框架。

4. 然后看 `scripted-quests`
   - 学数据驱动组件流水线。

5. 然后看 `QuestFramework`
   - 学任务链、触发器、观察器和投放机制。

6. 然后看 `StardewMods-QuestEssentials/QuestEssentials`
   - 学多阶段、多目标任务和 task 依赖激活。

7. 最后看 `Cataclysm-DDA`
   - 学开放世界动态目标生成、任务地点注入、任务链扩展。

8. `CustomNPCs_1.12.2-Unofficial`
   - 适合补看“多目标任务怎么抽象目标完成策略”。
