## 3. 核心主线

### 3.0 本附件绑定代码单元

本附件固定绑定：

- `U1 Prompt 资产正文与变量槽位合同`
- `U2 AI 服务方案 / 会话编排 / 输入组装`
- `U3 消息主档 + JSON sidecar 骨架`
- `U4 月度记忆压缩与长期经历回灌`
- `U5 行为协议 / 解析 / 修补 / 归一`
- `U6 群聊参与者 / 发言顺序 / 线程编排`
- `U7 传播协议与再入链`
- `U10 宿主 hook / UI 注入 / 最终写回`

第一阶段固定实现结论：

- `U1 = A`
- `U2 = B`
- `U3 = A`
- `U4 = B`
- `U5 = B`
- `U6 = B`
- `U7 = B`
- `U10 = C`

固定规则：

1. `U2 / U4 / U5 / U6 / U7` 第一阶段不允许 AFW 进入生产主链。
2. 这些单元第一阶段统一在 `Cloud` 执行参考复刻版。
3. `Runtime.Local` 与 `Runtime.<game> Adapter` 只做：
   - 结构修补
   - 白名单
   - 风险拦截
   - title-local 映射
4. 本地没有改语义的权力。

### 3.0.1 严格成功与两段提交

社交主链固定按严格成功口径运行：

1. `Cloud` 先写候选明文，但状态必须标明：
   - `未正式生效`
2. `Runtime.Local` 完成统一校验
3. `Game Mod` 真正显示 / 写回成功
4. 只有到这一步，`Cloud` 才能把：
   - 聊天
   - 记忆
   - 审计
   升级成正式正本

不允许：

1. `Cloud` 一生成结果就直接记成正式历史
2. 候选结果污染人物正式记忆
3. 本地落地失败但云端历史看起来像已经发生过

### 3.0.2 保密优先、快反馈、后台确认

当前社交主线固定采用：

1. `保密优先`
   - prompt 编排正文、记忆选取正文、规则链正文、完整 rendered prompt 只留在 `Cloud`
2. `快反馈`
   - 玩家先看到角色进入思考和开始出字
3. `后台确认`
   - 正式历史、正式记忆、宿主状态变更、commit promotion 在后面完成

固定规则：

1. 不为了少一次 `Cloud -> provider -> Cloud` 往返，就把最终 prompt 正文回传客户端。
2. 纯文本私聊、纯文本远程、thought 文字面允许先显示，再等 committed。
3. 物品、关系、承诺、事件、世界状态这类会改宿主状态的结果，不允许先演出来再补 committed。
4. 玩家前台看到的是：
   - 角色正在思考
   - 角色开始回答
   - 角色明确失败
   不是后台的审计、trace、commit 术语。
5. 等待和失败 copy 属于 title-local 玩家体验设计，不属于 shared runtime 术语。

### 3.1 私聊

参考锚点：

- `20_玩法功能/01_私聊功能.md`
- `10_共用系统/05_消息模型_通信通道与消息持久化.md`
- `10_共用系统/06_关系_记忆与摘要机制.md`
- `10_共用系统/07_行为协议_解析与执行.md`

固定落位：

1. `Cloud`
   - 选该游戏的私聊 prompt 资产
   - 编排最终提示词
   - 调 provider
   - 先写候选明文
   - 在宿主成功回执后，再升级聊天/记忆正本
2. `Runtime.Local`
   - 校验结构化结果
   - 拦非法动作
   - 生成统一归一结果
3. `Runtime.<game> Adapter`
   - 组装私聊事实包
   - 翻成该游戏可显示的执行清单
4. `Mod`
   - 取当前对话触发
   - 打开私聊宿主 UI
   - 显示文本 / 选项
   - 最终写回宿主历史与可见结果

固定补充：

1. 面对面私聊和远程一对一，共用同一条 actor-owned 聊天/记忆主骨架。
2. 区别只允许出现在：
   - carrier
   - availability 规则
   - UI 样式
3. 不允许各自长两套正式历史。

首次可见宿主：

- 游戏内私聊宿主

失败暴露点：

1. 游戏内私聊宿主文案
2. `Launcher -> 支持与帮助`

旧入口退役条件：

1. 旧的本地拼提示词路径必须退
2. 旧的本地直连 provider 路径必须退
3. 旧的绕过 `Runtime.Local` gate 的私聊成功路径必须退

### 3.2 动态选项

参考锚点：

- `20_玩法功能/01_私聊功能.md`
- `90_附录/01_已解密Prompt总表.md`

固定规则：

1. 选项由 Cloud 生成结构化结果。
2. `Runtime.Local` 负责检查数量、字段、合法性。
3. `Runtime.<game> Adapter` 负责翻成该游戏 UI 选项模型。
4. `Mod` 负责显示选项并把玩家选择回送下一轮。
5. 动态选项属于正式输出，不是前端装饰。

不允许：

1. Mod 本地自己临时补一套“假选项”成功路径。
2. 前端自己临时补一套“看起来差不多”的选项。
3. 选项结构不合格仍继续显示。

### 3.3 记忆压缩

参考锚点：

- `10_共用系统/06_关系_记忆与摘要机制.md`
- `20_玩法功能/01_私聊功能.md`

固定规则：

1. 记忆正本在 Cloud。
2. 压缩结果明文保存在 Cloud 审计面。
3. `Runtime.Local` 只消费摘要和回放所需投影。
4. `Runtime.<game> Adapter` 负责把记忆摘要接回当前游戏语义包。
5. `Mod` 负责把玩家可见记忆面显示出来。
6. 记忆只吃真正落地成功的结果。

首次可见宿主：

- 游戏内信息面板 / 记忆面

失败暴露点：

1. 游戏内记忆面失败文案
2. `Launcher -> 支持与帮助`

### 3.4 交易 / 给物 / 承诺

参考锚点：

- `20_玩法功能/08_交易与给物互动.md`
- `10_共用系统/07_行为协议_解析与执行.md`

固定规则：

1. Cloud 只给结构化建议，不直接改宿主。
2. `Runtime.Local` 必须经过动作白名单、参数合法性、phase-scoped gate。
3. `Runtime.<game> Adapter` 必须把被接受动作翻成该游戏自己的执行清单。
4. `Mod` 负责最终：
   - 发物
   - 改关系
   - 渲染物品 carrier
   - 回传执行回执
5. 交易 / 给物 / 承诺 / 接受 / 拒绝 属于社交主链里的正式动作协议。

首次可见宿主：

- 游戏内文本 carrier / 物品 carrier / 对话宿主

失败暴露点：

1. 游戏内 carrier / 对话失败文案
2. `Launcher -> 支持与帮助`

### 3.5 远程一对一

参考锚点：

- `20_玩法功能/04_传音与远程通信.md`
- `20_玩法功能/01_私聊功能.md`

固定规则：

1. 远程一对一与私聊共用同一条正式聊天/记忆骨架。
2. `Cloud` 负责远程建议与内容候选。
3. 最终能不能发，必须由：
   - `Runtime.<game> Adapter`
   - `Runtime.Local`
   根据宿主事实裁决。
4. 裁决事实包括：
   - 当前通道是否可用
   - 当前人物是否可达
   - 当前剧情 / 节日 / 宿主状态是否允许

### 3.6 群聊历史与回流

参考锚点：

- `20_玩法功能/02_群聊功能.md`
- `10_共用系统/05_消息模型_通信通道与消息持久化.md`

固定规则：

1. 群聊保留独立群历史。
2. 关键群聊内容必须回流到参与者个人后续判断。
3. 回流时必须带：
   - `来自群聊`
   的来源标记。
4. 不允许伪装成普通私聊。
5. 群聊发言顺序主要由 `Cloud` 编排。
6. 本地只做：
   - participant 集合校验
   - 线程校验
   - 旧回包拦截

### 3.7 信息传播

参考锚点：

- `20_玩法功能/05_信息裂变与社会传播.md`
- `10_共用系统/07_行为协议_解析与执行.md`

固定规则：

1. 信息传播属于社交主链里的独立协议分支。
2. 第一阶段按参考 mod 的 `ConveyMessage` 口径复刻。
3. 传播目标主要由 `Cloud` 编排选择。
4. 本地只做：
   - 目标合法性
   - 当前可达性
   - 当前通道是否可送
   - hop / 超限 / 冲突校验
5. 传播必须先落到通信载体，再影响第三方。
6. 不允许系统直接把结论写进第三方脑子。

## 4. 本批 AFW / 治理收口

### 4.1 命中的 profile 组合

- `backend-service + generic-interactive-client + repo-local governance overlay`

### 4.2 首次可见宿主

- 游戏内对话宿主
- 游戏内信息面板
- 游戏内物品 carrier / 对话 carrier
- 游戏内群聊宿主
- 游戏内远程线程宿主
- 游戏内目标传播宿主

### 4.3 补看宿主

- `Launcher -> 游戏页`
- `Launcher -> 支持与帮助`

### 4.4 资源来源真相源

1. 提示词资产：Cloud
2. 记忆正本：Cloud
3. 聊天正本：Cloud
4. 候选阶段明文：Cloud

### 4.5 输入真相源

1. 宿主事实：Mod
2. 事实包统一形状：Runtime.<game> Adapter
3. 本地 gate：Runtime.Local
4. 群聊顺序与传播目标语义：Cloud

### 4.6 authority owner

1. 私聊提示词 authority owner = Cloud
2. 记忆正本 owner = Cloud
3. 本地归一 owner = Runtime.Local
4. 最终宿主执行 owner = Mod
5. 群聊顺序与传播目标 owner = Cloud

### 4.7 生命周期 owner

1. 宿主触发 = Mod
2. 事实冻结 = Runtime.<game> Adapter
3. 云端建议 = Cloud
4. 本地归一 = Runtime.Local
5. 执行清单 = Runtime.<game> Adapter
6. 宿主提交 = Mod

### 4.8 并发 / 时序 contract

1. 一轮私聊只认一份冻结事实包
2. 切 NPC 后旧结果不得覆盖新面板
3. 旧请求回包只能入证据，不得冒充当前 committed
4. 群聊一轮只认一份冻结 participant 集合
5. 传播必须先落 carrier，再允许第三方再入判断

### 4.9 review 必查点

1. 旧私聊成功路径是否退役
2. 本地是否还在拼最终提示词
3. 记忆正本是否仍只在 Cloud
4. 物品和关系是否仍由 Mod 最终落地
5. 私聊与远程一对一是否仍共用同一 actor-owned 历史骨架
6. 群聊是否既保独立历史，又把关键内容带来源标记回流个人判断
7. 传播是否仍先过通信载体，而不是直接改第三方状态
