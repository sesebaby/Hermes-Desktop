# Stardew OpenAI 400 和手机消息 UI 访谈记录

## 背景

用户手动测试 Stardew NPC move 后发现两个问题：

1. OpenAI `400 Bad Request` 会影响 Hermes Agent 回复。
2. NPC 私聊和主动说话会打开全局对话框，打断玩家行动，也可能中断 Haley 这类 NPC 的 move。

上下文快照：`.omx/context/stardew-openai400-phone-ui-20260504T222112Z.md`

## 关键结论

- 玩家主动点击 NPC 时，继续走原版对话框。
- Hermes/NPC 主动发文本时，如果玩家和 NPC 在同一地图且距离不超过 8 格，显示 NPC 头顶气泡，不打开全局对话框。
- Hermes/NPC 主动发文本时，如果不在同一地图，或同地图距离超过 8 格，进入右侧手机/微信式消息。
- “视野内”首版只按 8 格距离算，不考虑 NPC 朝向。
- 手机 UI 首版必须能回复 NPC，不能只做收件箱。
- 手机回复必须进入 Hermes Agent，按“给对应 NPC 的私聊消息”处理。
- 手机 UI 采用更贴近微信的样式：联系人列表、聊天线程、底部输入框。
- 手机关闭时不打断玩家行动；手机打开输入时可以短暂占用键盘，关闭后恢复正常游玩。
- 优先级：先修 OpenAI 400，再做文本展示分流，最后做手机 UI 和回复进 Agent。
- 用户追加要求：方案中关键节点必须添加日志，方便后续检查问题。

## 用户确认的不做

- 首版不做跨存档/重启后的消息持久保存。
- 首版不做表情、图片、语音、群聊、搜索等复杂微信功能。
- 首版不做真实手机打开动画，只做右侧入口轻微震动/闪烁。
- 不改变玩家主动点击 NPC 的原版对话逻辑。
- 不直接永久清 NPC 日程；`hold/控制租约` 仍作为移动可靠性的待解决问题单独处理。
- 不把所有原版游戏对话都改成手机消息，只处理 Hermes/NPC 主动触发的文本。

## 访谈轮次

1. 先确认哪些行为应该避开全局对话框。
2. 用户确认：主动点击走原版；近距离 NPC 主动文本走头顶气泡；远距离走手机消息。
3. 用户确认：视野内只按 8 格距离，不看朝向。
4. 用户要求：手机首版必须能回复 NPC。
5. 用户接受微信式布局，并接受打开手机输入时短暂占用键盘。
6. 用户确认：手机回复必须进入 Hermes Agent。
7. 用户同意优先级：OpenAI 400 -> 文本展示分流 -> 手机 UI。
8. 用户接受首版 non-goals。
9. 用户追加要求：关键节点必须有日志。
