# 延迟修复参考实现映射合同

## 这份合同管什么

本轮只管“AI 回复很慢”的上下文压缩、总结、记忆注入、模型调用耗时拆分问题。

本轮不管：

- 不重做整个 NPC 自主循环。
- 不新增第二套 memory、persona summary、NPC brain 或隐藏 fallback。
- 不用“直接关 summary / 把 interval 调很大”冒充修复。
- 不把 Stardew 宿主改成替 NPC 决策的层。

## 参考实现主链复原

### 1. Trigger

参考项目中，压缩由 `ContextEngine` 负责判断，主要看真实或预估 request tokens 是否超过阈值。不是只要有旧消息被挤出，就立刻总结。

当前项目中，`ContextManager` 先按固定 recent window 拆出 `evictedMessages`；只要存在 evicted messages，并且 summary 为空、过期或压力高，就可能在回复前总结。

### 2. Snapshot

参考项目把完整 messages 交给 compressor，再保护 head 和 tail，中段才进入压缩。

当前项目先固定保留最近 6 个 user turn，旧消息直接成为 summary 输入；总结输入还会截断到最新 4000 字符。

### 3. Prompt / Summary Assembly

参考项目生成结构化 handoff summary，并通过 `call_llm(task: "compression")` 走压缩专用辅助路由。

当前项目用较短模板总结 evicted messages，并调用主 `IChatClient.CompleteAsync`。这会把“回答前的整理工作”和“真正回答”串在同一条模型链路上。

### 4. Parse / Repair / Normalize

参考项目在 LLM 总结前先 cheap prune 旧 tool results，保护 tool_call/tool_result 边界，并给 summary 加 reference-only 前缀。

当前项目 inspected path 没看到同等级的旧工具输出预裁剪、tool pair 边界保护和 reference-only summary prefix。

### 5. Projector / Executor

参考项目压缩后返回新的 OpenAI-format messages，插入或合并 summary，必要时重建 system prompt，并处理 role alternation。

当前项目把 summary 放进 `SessionState`，再由 `PromptBuilder` 作为 system-layer session state 每轮注入。

### 6. Authoritative Writeback

参考项目在压缩前通知 memory provider，随后走 session 压缩/分裂/写回路径。

当前项目有 memory/plugin pre-compress hook，但 summary 主要写入 `SessionState`；是否跨进程持久化还不能按参考对齐来声称。

### 7. Player-visible Surface

参考项目里压缩是内部上下文空间维护，用户只看到最终回答。

当前用户看到的是“回复前卡很久”。日志证据显示 2026-05-07 13:42:09 开始处理 Penny，13:42:24 才完成 summary，13:42:45 收到决策回复；其中 summary 前置消耗约 14.8 秒。

## 当前项目映射位

推荐第一版落在 `ContextManager` 这一层：先修 trigger 和诊断，把“前置总结”从默认路径降级为“真实 token 压力才触发”。

不推荐第一版直接完整迁移 Python `ContextEngine`，因为那会同时碰：

- 生命周期接口；
- session 压缩写回；
- auxiliary compression 配置；
- tool-result pruning；
- role alternation；
- 现有 `CompactionSystem` 去留；
- NPC runtime 和桌面主 chat 共用边界。

这会把一个明确的延迟问题扩大成上下文系统重构。

## 允许偏离

允许第一版用 C# 本地结构做“最小参考对齐”：

- 先不新增完整 `IContextEngine` 抽象。
- 先不启用 session split/writeback。
- 先不添加新的外部依赖。
- 先补诊断日志和更正确的 trigger。

允许的理由：这属于 title-local adaptation，目标是先消除最明显的每轮前置 LLM 总结税，同时保留后续完整迁移空间。

## 禁止偏离

以下做法会破坏参考优先：

- 直接关闭 summary，却不说明上下文丢失代价。
- 只把 stale threshold 调大，却不修 trigger 语义。
- 新增第二套 NPC summary/memory/persona lane。
- 让宿主根据文本替 NPC 推断行动。
- 保留两个同时可走的压缩主路径，形成影子实现。
- 声称“已对齐参考项目”，但没有 trigger、snapshot、summary assembly、writeback 的逐项证据。
