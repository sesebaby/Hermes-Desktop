# 星露谷 NPC 世界背景与地点候选上下文

## 任务陈述
按“通用世界背景 + Haley 个人地点偏好 + 地点候选工具事实”增强 Stardew NPC 自治移动，让 Haley 能基于世界知识和当前可去地点主动选择有意义的移动。

## 期望结果
- NPC 仍由 agent 主动决策，host 只提供观察事实、技能说明和工具执行。
- Haley 的 prompt 中包含通用 Stardew 世界背景和个人地点偏好。
- 通用世界背景按 skill-creator 渐进式披露设计：主 skill 精简，详细地点资料放 reference。
- 首轮 `ObserveAsync` 事实包含机器可读 `placeCandidate`，不是只给相邻 `moveCandidate`。
- `stardew_move` 工具允许使用最新 observation 中的 `moveCandidate` 或 `placeCandidate`。

## 已知事实
- `StardewNpcAutonomyPromptSupplementBuilder` 会读取 NPC pack 的 `skills.json` required，并把 `skills/gaming/{skill}.md` 拼进 system prompt。
- Haley pack 位于 `src/game/stardew/personas/haley/default`。
- 当前 `StardewQueryService.BuildStatusFacts` 已将 `moveCandidate` 映射为 observation facts。
- Bridge status 由 `Mods/StardewHermesBridge/Bridge/BridgeHttpHost.cs` 生成。
- 既有原则：除私聊外，不允许事件驱动 NPC move；行为变化应通过 prompt/skill/观察事实/工具描述完成。

## 约束
- 不新增第二套 NPC agent 链。
- 不强制长期任务结构。
- 不让 host 选择 Haley 的目的地。
- 本轮优先提供能被当前 bridge 执行的地点候选；跨 location 复杂寻路留作后续。
