# 星露谷 NPC 闲置微动作上下文快照

- 创建时间：2026-05-07T07:23:16Z
- 模式：deep-interview / standard
- 任务陈述：用户希望在 NPC 发呆、站在家里或固定地点时，让本地小模型驱动不干涉原版日程/主 agent 日程的轻量动作，例如头顶表情、左右转头、原地动作，以增强沉浸感。
- 期望结果：形成一个可规划的需求规格，明确本地小模型在“闲置微动作”中的作用、边界、参考 mod、工具/skill 形态、验收标准。
- 已述方案：给本地小模型专门的 skill 和工具，只允许它做低风险的视觉/姿态微动作，不接管移动、日程、人格主决策。
- 意图假设：把当前“本地小模型作用有限”的弱点转成优势，让它承担低风险、高频、低成本的环境鲜活度任务。
- 代码事实：
  - 当前 bridge 已有 `NpcOverheadBubbleOverlay`，能在 NPC 头顶绘制文字气泡：`Mods/StardewHermesBridge/Ui/NpcOverheadBubbleOverlay.cs`。
  - 当前 bridge 已能调用 `npc.faceDirection(...)`、`npc.animateInFacingDirection(...)`，但主要用于移动执行：`Mods/StardewHermesBridge/Bridge/BridgeCommandQueue.cs`。
  - 当前 local executor 支持 `observe` / `task_status` / `move` / `wait` / `escalate`，机械 move target 会走 `host_deterministic` 而非本地模型。
- 本地参考发现：
  - `VillagerAgent` 参考资料里有 idle prompt、emoji、murmur、performMovement 概念，适合借鉴“低风险动作意图格式”，但不是 Stardew SMAPI 直接实现。
  - `The Stardew Squad` 本地源码已有 idle animation 池、`IdleTicks`、概率触发、cooldown、`Data/animationDescriptions` 解析、动画结束回标准 idle frame；适合参考短 idle animation 机制。
  - `CustomCompanions` 本地源码已有 `HOVER`、`WANDER`、`JUMPER` idle behavior、`jumpWithoutSound(...)`、hover/jump/fidget 运行方式；第一版只借原地 fidget，不借会改 tile 的 wander。
  - `Idle Plus Walk` 公开描述是真实 idle pose / animation logic 参考。
  - `Dialogue Emotes`、`Think-n-Talk`、`The Living Valley`、`Social Valley` 主要是气泡、聊天、AI 对话，不是身体动画系统。
  - `NPCGeometry` 借渲染几何参数，不当动作系统参考。
- 在线初步发现：
  - Nexus `Interactive Emotes`：玩家 emote 附近 NPC/宠物/马实时回应 emote，说明 NPC emote reaction 方向有现成 mod。
  - Nexus `Dialogue Emotes`：给 NPC 对话增加头顶 emote bubble 支持。
  - CurseForge `Idle Plus Walk`：修改 NPC animation logic，让角色 mod 支持 unique idle poses。
  - CurseForge `NPC Adventures`：followers 有 idle animations 和战斗 speech bubbles。
  - Stardew Wiki `Modding:NPC data` 有 `CanGreetNearbyCharacters`、`DumpsterDiveEmote`、`HiddenProfileEmoteStartFrame` 等 NPC data 字段，可作为原版数据能力线索。
- 约束：
  - 不干涉原版日程。
  - 不干涉主 agent 的任务/移动日程。
  - 本地无 tmux；后续不要依赖 tmux-only 流程。
  - 深访阶段不直接实现。
