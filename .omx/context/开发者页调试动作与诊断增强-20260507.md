# 开发者页调试动作与诊断增强上下文快照

## 任务陈述

在现有开发者页一期只读观察台基础上，规划后续增量：

- 增加一个短测试命令，能把海莉或潘妮放到 `Town`，便于手测 NPC 从镇上主动跨地图去 `Beach`。
- 第二阶段把受控调试动作迁移到开发者页。
- 第三阶段增加追踪筛选、诊断包导出、上下文注入差异视图。

## 期望结果

形成一份可执行的中文方案文档和测试规格，后续可交给 `$ralph` 或 `$team` 实现。当前阶段只规划，不修改产品代码。

## 已知事实

- 现有开发者页已接入导航和只读观察台：`Desktop/HermesDesktop/MainWindow.xaml.cs` 已有 `developer` 映射，`Desktop/HermesDesktop/Views/DeveloperPage.xaml.cs` 使用 `NpcRuntimeWorkspaceService`、`NpcRuntimeSupervisor` 和 `NpcDeveloperInspectorService`。
- `Dashboard` 与 `Agent > Runtime` 已有跳转到开发者观察台的入口。
- 现有 `Agent > Runtime` 仍保留固定调试按钮：`StardewNpcDebugActionService.SpeakAsync(...)` 和 `StardewAutonomyTickDebugService.RunOneTickAsync(...)`。
- 现有 SMAPI 控制台命令 `teleport <海莉|haley|潘妮|penny>` 位于 `Mods/StardewHermesBridge/Commands/TestTeleportCommand.cs`，实际行为是把玩家传送到 NPC 附近，不是把 NPC 放到 `Town`。
- 跨地图最小探针 PRD 要求手测时 NPC 在 `Town`，目标是 `Beach` 的明确坐标，mechanical target 由宿主确定性执行 `stardew_navigate_to_tile`，并观察 `routeProbe`。
- 开发者页一期 PRD 明确首版只读；真实移动、说话、私聊、自然语言目标输入等有副作用动作留到后续阶段。

## 约束

- 新建方案、计划、测试规格、上下文快照等文档统一使用中文文件名，并带日期。
- UI 普通文案使用中文；日志、JSON、路径、工具名、模型原文等原始内容可保持原文。
- 复用现有后端基础设施，不新增第二套 NPC runtime、任务、待办、记忆、transcript、trace 或工具通道。
- 短测试命令必须明确是手测辅助，不能冒充自然移动成功。
- 受控调试动作必须有明显副作用边界、确认/结果反馈和审计日志。
- 诊断包导出不能泄露 bridge token、API key、凭据或不必要的隐私内容。
- 计划若涉及 `Desktop/HermesDesktop/**`，执行阶段必须遵守桌面子项目 AGENTS 和 `.github/instructions/*.instructions.md`。

## 未决问题

- 短测试命令的最终命名应避免覆盖现有 `teleport`，建议规划为专用命令，例如 `hermes_npc_town <海莉|haley|潘妮|penny>`。
- 桌面开发者页是否直接暴露“放到镇上”按钮，还是第二阶段只先迁移已有“说话 / 单步唤醒”，把 NPC 位置重置作为桥接层控制台命令保留。
- 上下文注入差异视图应展示“原始 prompt 包 vs 预算裁剪后包”，还是展示“最近两次同 NPC 调用的上下文差异”。
