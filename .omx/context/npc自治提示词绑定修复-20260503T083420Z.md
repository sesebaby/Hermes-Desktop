# NPC 自治提示词绑定修复

## 任务陈述

为 Stardew NPC 常驻自治补齐 persona pack 与 bundled Stardew 行为技能的 runtime/prompt 注入链路，解决 Haley 能主动私聊但未稳定自主 move 的业务逻辑缺口。

## 期望结果

- `NpcAutonomyLoop` 继续保持“每 tick 观察事实并交给 agent 自主决策”，不增加事件驱动移动。
- NPC autonomy agent 的稳定系统提示词必须包含 persona pack 的 `facts.md`、`voice.md`、`boundaries.md` 以及 `skills.json.required` 声明的 Stardew 行为技能正文。
- `skills.json` 不再只是 pack 完整性文件，而是正式参与 runtime autonomy binding。
- 修复要通过测试证明 Haley/Penny autonomy prompt 真的收到人格与行为技能内容。

## 已知事实 / 证据

- `src/runtime/NpcAutonomyLoop.cs` 当前只记录 observation/event facts，然后调用 agent；直接消息只有“Use these passive game facts to decide the next autonomous step”，事件只是上下文。
- `src/runtime/NpcRuntimeContextFactory.cs` 当前只注入通用 Stardew runtime supplement 与通用 `SkillManager.BuildSkillsMandatoryPrompt()`。
- `src/runtime/NpcRuntimeSupervisor.cs` 的 private chat handle 支持 `SystemPromptSupplement`，但 autonomy handle 当前传入 `systemPromptSupplement: null`。
- `src/runtime/NpcRuntimeBindings.cs` 的 `NpcRuntimeAutonomyBindingRequest` 当前没有 `SystemPromptSupplement` 字段。
- `src/runtime/NpcNamespace.cs` 会把 persona pack 复制到 runtime namespace，并把 `SOUL.md` 种进去，但生产链路没有把 `facts.md` / `voice.md` / `boundaries.md` / `skills.json` 拼进 autonomy prompt。
- `src/game/stardew/personas/haley/default/skills.json` 和 `src/game/stardew/personas/penny/default/skills.json` 都声明 required：`stardew-core`、`stardew-social`、`stardew-navigation`。
- `skills/gaming/stardew-core.md` 明确要求 observe-decide-act，且动作只能通过注册 Stardew tools。
- `skills/gaming/stardew-navigation.md` 明确 move 要走 `move` task contract，并跟踪 command/status。
- 通用 `SkillManager` 只扫描 `SKILL.md` 目录型技能；`skills/gaming/stardew-*.md` 是普通 markdown，不会自然进入 `BuildSkillsMandatoryPrompt()`。
- `stardew_move` 工具、`StardewCommandService` 与 bridge `/task/move` 路径已存在；当前更像 prompt/业务指导缺失，而不是工具链完全断开。
- bridge 端现有 move 是 `setTilePosition` 一类的瞬移式位置变更，不是可见 walking animation；这属于后续表现层问题，不应混进本次 autonomy prompt 修复。

## 约束

- 计划名称和内容必须用中文。
- 不允许把 NPC 行为改成事件驱动；当前除了私聊以外，不允许任何事件方式驱动 NPC。
- 如需调整 NPC 行为，只能通过提示词/persona/skill guidance，让 agent 自己决定下一步。
- 不让 host/background service 代替 NPC 决策具体移动目标或动作。
- 不新增第二套桌面 agent / NPC agent 能力装配体系；继续复用 `AgentCapabilityAssembler` 与共享 runtime。
- 不把 Stardew skill 正文硬编码进 C# 字符串；应从仓库现有 markdown 文件读取。
- 不在本计划内解决 visible walking animation，除非后续单独立项。

## 未知 / 开放问题

- bundled Stardew skill markdown 的根目录解析应以 workspace root、app base directory，还是 pack source provider 为主；需要选择一个测试可控且运行时稳定的定位策略。
- `skills.json.optional` 是否本次也注入；初步倾向 required 必须注入，optional 可在存在时注入但缺失不阻塞。
- 缺少 required skill 文件时应 fail-fast 还是降级；初步倾向 fail-fast，因为 silent fallback 会再次造成“看似完成但 NPC 不具备能力”。
- persona `facts/voice/boundaries` 是否同时用于 private chat；本次优先锁 autonomy，private chat 是否复用可作为 follow-up。
